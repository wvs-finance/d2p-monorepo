## Component & Layout Contract (net-new)

### 1. Pipeline-trace (centerpiece) ‚Äî RECOMMENDATION: **vertical stepper**, not a free-form flow diagram

**Decision (recorded as contract, marked recommendation):** Use a **vertical stepper** ‚Äî a top-to-bottom ordered sequence of stages connected by a 1px `border-l` rail with a stage marker per node. Rationale over a horizontal flow/graph diagram:
- Mobile-first at 360px: a horizontal node-graph cannot fit 6 stages without horizontal scroll or SVG zoom (LCP/`impeccable` risk); a vertical stepper reflows natively.
- It is RSC-renderable with plain DOM (no client SVG layout lib, no recharts) ‚Äî keeps the surface in the no-JS read-first contract and out of the lazy-recharts bundle.
- The pipeline IS linear and deterministic (print ‚Üí prompt ‚Üí action leg ‚Üí size leg ‚Üí decision ‚Üí illustrative position); a stepper is the honest visual grammar for a deterministic sequence. A "flow diagram" implies branching/agency the deterministic `pure` pipeline does not have.

**Stages (exactly 6, equal visual weight ‚Äî same marker, same title size, same card shell):**
| # | Stage | Content | Provenance |
|---|-------|---------|------------|
| 1 | Macro print | `macroValue` (mono), `dataKey` label, captured date (em-dash for observed) | `testnet-agent` pill |
| 2 | Built prompt (deterministic) | The `_buildPrompt(actual, consensus)` reconstruction in a `<pre>` mono block; caption: "reconstructed deterministically from actual + operator-supplied consensus" | `testnet-agent` |
| 3 | Action leg (Qwen3-30B, temp 0) | decoded `action` enum badge (reuse `HedgeDecisionCard` ActionBadge neutral shell); model id `12847293847561029384` shown as mono caption; leg = `Action` | `testnet-agent` |
| 4 | Size leg (Qwen3-30B, temp 0) | decoded `sizeBps` (mono); leg = `Size` | `testnet-agent` |
| 5 | Decision | action + sizeBps + surprise (surprise gated behind operator-supplied caveat, reuse Phase-6 rule) | `testnet-agent` |
| 6 | Illustrative position | `sizeBps`‚Üífraction-of-max mapping (real arithmetic), rendered as a percentage/fraction (e.g. "6800 bps ‚Üí 0.68 of max") ‚Äî **NEVER a `$` figure**; caption: "illustrative ‚Äî not an executed position" | links to the position panel (¬ß2) |

`SYSTEM_PROMPT` viewer: a single **collapsible** disclosure ABOVE or adjacent to stage 2 ‚Äî accent-text trigger ("View system prompt" / "Ver prompt del sistema") + caret, expanding to a `<pre>` mono block. Default collapsed (it is long); the BUILT prompt (stage 2) stays expanded. (Note: this collapse is permitted because it is a verbatim long constant, NOT a disposition/verdict ‚Äî the anti-fishing no-collapse rule from CROSS-09/LAB-05 applies to FAIL/PASS narratives and disposition memos, which this is not.)

No fabricated chain-of-thought anywhere; copy says "the deterministic decision pipeline", never "the agent's reasoning/thoughts".

### 2. Position-execution panel ‚Äî not-deployed empty state

- Panel shell matches `HedgeDecisionCard`: `rounded-lg border border-border-default bg-bg-surface p-4 space-y-4`.
- Header: `fork-verified / not-live` pill (¬ß new tier) + heading "Posici√≥n (LongGammaWrapper)" / "Position (LongGammaWrapper)".
- Body: the typed `WrapperPositionView` fields rendered as a `<dl>` row list (same idiom as `HedgeDecisionCard`), every value showing **em-dash `‚Äî`**. Fields surfaced (per spec ¬ß2/¬ß5 `adaptWrapper` rules ‚Äî display labels only; no value computed in Phase 7): position legs/health (composed via `pool()`), surviving collateral (via `ct0()/ct1()` `convertToAssets`), `positionTokenId`, residual. NEVER surface `lastSurviving0/1` or `deposited0/1` as "current"; NO `realizedCosts` field exists.
- A single honest caption directly under the heading: "No desplegado ‚Äî verificado en fork, sin posici√≥n en cadena." / "Not deployed ‚Äî fork-verified, no on-chain position." (`text-sm text-text-muted`).
- No `$`, no fabricated number, no green token anywhere in this panel.

### 3. Disabled management controls ‚Äî RECOMMENDATION: **visible buttons + persistent inline caption** (NOT tooltip-only)

**Decision (recorded as contract, marked recommendation):** Render close / claim / agent as **real `<button disabled>` elements** with a **persistent inline caption** beneath the control group ‚Äî NOT a hover/focus tooltip as the sole disclosure.

Rationale: a tooltip is not perceivable on touch (no hover) and is unreliable for screen readers as the *only* explanation ‚Äî the honesty invariant requires the not-live reason to be perceivable beyond color AND beyond hover. The inline caption is always rendered, always read.

Disabled-state visual contract (perceivable beyond color ‚Äî a11y):
- Native `disabled` attribute set (keyboard-unfocusable, conveyed to AT).
- `aria-disabled="true"` + `aria-describedby` pointing at the inline caption id.
- Visual: `text-text-muted border-border-default bg-bg-surface cursor-not-allowed opacity-60` ‚Äî and a **non-color cue**: a `lucide-react` `Lock` icon inside each button (color + icon + text). Reduced opacity alone is NOT sufficient; the lock icon + caption carry the meaning.
- Buttons use the neutral pill/button shell (NOT accent fill) ‚Äî they are not primary CTAs this phase.
- Inline caption (one per group, `role` default, referenced by all three buttons): "No disponible ‚Äî fork-verificado, no desplegado. Sin transacci√≥n." / "Unavailable ‚Äî fork-verified, not deployed. No transaction."
- Rendered DOM MUST NOT contain "executed/realized/placed/ejecutad/realizad" (e2e honesty grep).

### 4. Liveness pill ‚Äî client island

- Reuses the `WalletStatusPill` CROSS-09 shell verbatim (`inline-flex ‚Ä¶ rounded-full px-2.5 py-1 text-sm font-normal ring-1 ring-inset`).
- Only `snapshot` (default) and `polling` (`SOMNIA_LIVE`) render in Phase 7 (colors/icons above). The `LivenessSource<T>` `useSyncExternalStore` contract drives it; on snapshot the subscribe is an immediate no-op unsub.
- Placed in the page header near the route title, not inside the trace, so it describes the whole surface's freshness.

### 5. Master‚Äìdetail link (Phase-6 feed ‚Üí detail)

- Each `HedgeDecisionCard` (or its container on `/apps/abrigo/agent`) gains a link to `/apps/abrigo/agent/[id]`.
- Affordance: `text-accent-text hover:text-accent-hover` label "Ver la traza de decisi√≥n" / "View decision trace" + a `lucide-react` `ChevronRight` icon + underline-on-hover. CROSS-09: color is NOT the sole link signal (icon + underline present).
- The whole card MAY be a link, but the equal-weight invariant (all four actions identical) must be preserved ‚Äî the link affordance is identical on every card regardless of action.
