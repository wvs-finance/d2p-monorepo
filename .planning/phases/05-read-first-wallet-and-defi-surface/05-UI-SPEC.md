### RainbowKit Theme — HEX Values Required (BLOCKER-3 corrected)

RainbowKit alpha-composites `accentColor` and derived states internally. oklch strings break the compositor. Pass HEX approximations to properties RainbowKit manipulates; use `var(--token)` ONLY for properties RainbowKit renders verbatim (modal surface CSS vars).

```typescript
// In app/(defi)/providers.tsx
import { lightTheme } from '@rainbow-me/rainbowkit'

const rbkLightTheme = lightTheme({
  accentColor: '#a87c3a',         // HEX — RainbowKit compositor; approx of oklch(0.6 0.08 70)
  accentColorForeground: '#f8f5f0', // HEX — approx of --bg-canvas oklch(0.97 0.005 80)
  borderRadius: 'medium',          // 0.5rem matches --radius
  fontStack: 'system',             // overridden below via CSS vars
  overlayBlur: 'none',
})

// CSS var overrides — rendered verbatim by RainbowKit (safe to use var()):
// --rk-fonts-body        → var(--font-plex-sans), system-ui, sans-serif
// --rk-colors-modalBackground       → var(--bg-elevated)
// --rk-colors-modalBorder           → var(--border-default)
// --rk-colors-modalText             → var(--text-primary)
// --rk-colors-modalTextSecondary    → var(--text-secondary)
// --rk-colors-actionButtonBorder    → var(--border-default)
// --rk-colors-connectButtonBackground → '#a87c3a'  ← HEX (compositor applies alpha)
// --rk-colors-connectButtonText       → '#f8f5f0'  ← HEX
```

## Route / Provider Boundary (BLOCKER-2 — FOUND-11 clean)

The per-instrument route URL is `/apps/abrigo/instruments/{id}/{chain}`.

Route groups in Next.js are URL-transparent. The instrument detail pages MUST be placed at `app/(defi)/apps/abrigo/instruments/[id]/[chain]/page.tsx` — NOT under `app/(apps)/`. This yields the same public URL `/apps/abrigo/instruments/{id}/{chain}` while keeping the pages physically inside the `(defi)` route group, which is wrapped by `WagmiProvider` + `RainbowKitProvider`. This is mandatory: `useAccount`, `useChainId`, `useSwitchChain` throw at runtime unless the component tree is inside the Wagmi context.

The instruments index (`/apps/abrigo/instruments`) also lives at `app/(defi)/apps/abrigo/instruments/page.tsx`.

The existing `app/(apps)/apps/abrigo/dashboard/` stays untouched. Wallet JS must not leak into `(lab)` or `(apps)` (FOUND-11 enforced by layout boundary).

```
app/
  (apps)/apps/abrigo/dashboard/    ← existing, untouched
  (defi)/
    layout.tsx                     ← existing, wraps WagmiProvider/RainbowKitProvider
    providers.tsx                  ← Phase 5 replaces placeholder
    apps/abrigo/
      instruments/
        page.tsx                   ← instruments index (honest empty)
        [id]/[chain]/
          page.tsx                 ← per-instrument page (payoff + pool + wallet)
```

## Wallet State Machine (BLOCKER-4 corrected)

Source: `wagmi getAccount` return values. Exactly 4 reachable states via EVM connectors.

| State | Derivation from wagmi `useAccount` | Reachable? |
|-------|-------------------------------------|-----------|
| DISCONNECTED | `status === 'disconnected'` | YES |
| CONNECTING | `status === 'connecting' \|\| status === 'reconnecting'` | YES |
| CONNECTED_WRONG_CHAIN | `status === 'connected' && chain === undefined` — wagmi returns `chain: undefined` when the connected EVM chain is NOT in our 5 configured chains | YES |
| CONNECTED_READY | `status === 'connected' && chain !== undefined` — chain is one of our 5 (celo/mainnet/base/arbitrum/optimism) | YES |

**5 configured chains (from `lib/wagmi/config.ts`):** celo, mainnet (Ethereum), base, arbitrum, optimism. NOT Polygon, NOT BNB.

**TRACKED WAIVER — non-EVM wallets (BLOCKER-4):** A Solana wallet or any other non-EVM wallet CANNOT connect through wagmi EVM connectors. There is no 5th "UNSUPPORTED_CHAIN" state reachable through this stack. The prior spec's `UNSUPPORTED_CHAIN` state (with Solana example and `XCircle` pill) is UNREACHABLE via wagmi EVM connectors. This state is removed from the 4-state machine. The DEFI-07 copy for "non-EVM networks like Solana" is also removed. The 4-state count in this spec matches DEFI-02 honestly. Future non-EVM wallet support (e.g. via a Solana adapter) is deferred.

**CONNECTED_WRONG_CHAIN example copy (DEFI-07):**
- es-CO: "Estás en [chain]. Abrigo opera en Celo, Ethereum, Base, Arbitrum y Optimism."
- en: "You're on [chain]. Abrigo operates on Celo, Ethereum, Base, Arbitrum, and Optimism."
