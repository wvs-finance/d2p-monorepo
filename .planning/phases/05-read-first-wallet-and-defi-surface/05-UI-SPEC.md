## Spacing Scale

Multiples of 4 throughout. Source: existing Tailwind v4 `@theme` in `globals.css` + panoptic.xyz editorial-DeFi reference from `visual_design_reference.md`.

| Token | Value | Usage in Phase 5 |
|-------|-------|------------------|
| xs | 4px | Icon-to-label gaps within wallet-state pill; inline param value gaps |
| sm | 8px | Compact param row padding; wallet panel internal spacing |
| md | 16px | Default element spacing; risk callout internal padding (16px horizontal, 12px vertical) |
| lg | 24px | Card/panel internal padding; payoff diagram margin from container edge |
| xl | 32px | Section gaps on instrument page (params → diagram → pool panel) |
| 2xl | 48px | Major section breaks (risk callout → params block separation) |
| 3xl | 64px | Page-level vertical rhythm above the fold |
| hero | 120px (`--spacing-5xl`) | Section padding matching existing `globals.css` token (editorial-DeFi register) |

**Exceptions:**
- Wallet connect button touch target: minimum 44px height (WCAG 2.5.5 AA, DEFI-06) — use `h-11` (44px) on the inline connect prompt.
- Risk callout vertical padding at 360px viewport: 12px top/bottom (3px tighter than default md) to ensure disclosure stays above fold on smallest supported width.
- Payoff diagram minimum height: 240px at 360px viewport; 320px at 768px+. Not a spacing token — a layout constraint.

## Typography

Font: IBM Plex Sans (sans) + IBM Plex Mono (mono only for numeric values and chain addresses). No other typefaces. Source: `globals.css` + `visual_design_reference.md` (locked).

| Role | Size | Weight | Line Height | Tailwind class | Usage in Phase 5 |
|------|------|--------|-------------|----------------|-----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base font-normal leading-relaxed` | Risk disclosure body; wallet state explanatory text; instrument parameter descriptions |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium leading-snug` | Param row labels; wallet state labels; axis tick labels on payoff diagram; pool-state metric labels |
| Heading (card/section) | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` | Instrument name; section headings (Parameters, Pool State, Your Position); risk callout heading |
| Display (page h1) | 28px | 600 (semibold) | 1.2 | `text-[28px] font-semibold leading-tight` | Instruments index page h1; per-instrument page h1 |

**Mono usage (IBM Plex Mono only):**
- Numeric param values (strike price, slope): `font-mono text-sm font-normal`
- Chain addresses (if shown): `font-mono text-xs font-normal text-text-muted`
- Pool balance figures: `font-mono text-base font-medium`

**Weights in use: 400 (regular) and 600 (semibold).** 500 (medium) is used only for labels — kept as close to the "2 weights" constraint as the DeFi labelling density requires. No 700/bold on user-facing content.

### Accent Reserved For (exhaustive list — nothing else)

1. Risk callout border — 1px full hairline border (all 4 sides, NOT one-sided — impeccable gate) in `border-accent-default`
2. Risk callout heading text — uses `--accent-text: oklch(0.48 0.08 70)` (AA-safe small text variant)
3. Payoff curve line stroke — `stroke: var(--accent-default)` on the CFMM curve SVG path
4. Current price marker — vertical dashed line in `stroke: var(--accent-default)` on the payoff diagram
5. Wallet connect prompt link/CTA text — `text-accent-text` (AA-safe variant; NOT `text-accent-default` which fails 4.5:1 at small sizes)
6. Chain-switch CTA button — `bg-accent-default text-bg-canvas` (large enough for 3:1; confirm it renders ≥24px or ≥18.66px bold)
7. Focus ring — `--ring: var(--accent-default)` already wired via `focus-visible:ring-ring`
8. Active/selected state on instrument page navigation (future) — `border-b-2 border-accent-default`

**Accent-text rule (from visual_design_reference.md):** `--accent-default` (0.6 L) is only ~3.33:1 on `bg-surface` — sub-AA for text < 18.66px. All ochre text at < 18.66px (non-bold) or < 24px (bold) MUST use `--accent-text: oklch(0.48 0.08 70)` (`text-accent-text`). Fills, borders, large headings, and SVG strokes may use `--accent-default`.

## Page Layouts

### (a) Instruments Index (`/apps/abrigo/instruments`) — Honest Empty State

```
[page h1: "Instrumentos Abrigo" / "Abrigo Instruments"]
[empty state: PackageSearch icon (48px, text-muted) + heading + body + optional GitHub link]
```

- Canvas background (`--bg-canvas`). No cards, no grid. Centered, vertically in the upper viewport third.
- Max-width container: 1200px (matching editorial-DeFi register from visual_design_reference.md).
- Horizontal padding: 16px at 360px; 32px at 768px+.
- Empty state icon: `PackageSearch` from lucide-react at 48px. Color: `text-text-muted`. No illustrative diagram.
- **When instruments exist (post-deploy):** Render one card per instrument with: instrument name, chain badge, deploy date, "Ver instrumento" / "View instrument" link button. Cards use `bg-elevated` + `border-border-default`. No card-inside-card.

### (b) Per-Instrument Page (`/apps/abrigo/instruments/{id}/{chain}`) — Responsive from 360px

Layout order (top to bottom, all viewport widths from 360px):

```
1. [page h1: instrument name — 28px semibold]
2. [RiskCallout — full width, border-accent-default hairline, 12-16px padding, NOT dismissible]
3. [InstrumentParams — param grid/table]
4. [PayoffDiagram — client island, min-h-[240px], bg-surface container]
5. [PoolStatePanel — metric tiles, bg-surface, em-dash for null values]
6. [WalletPanel — DISCONNECTED state by default; per-instrument, not global]
```

**Single-column (360px–767px):**
- All 6 sections stack full width.
- Section gaps: 32px (`xl` spacing) between major blocks; 24px (`lg`) within subsections.
- RiskCallout at 360px: 16px horizontal padding, 12px vertical. Full text must be readable without horizontal scroll.

**Two-column (768px+):**
- Columns: 60% content (params + diagram) / 40% sidebar (pool state + wallet panel).
- RiskCallout spans full width above the column split.
- Gap between columns: 32px.

**h1 + RiskCallout above fold contract:** At 360px the h1 (28px × 1.2 line-height = ~34px), a 16px gap, and the RiskCallout (heading ~20px + body ~48px + padding 24px = ~92px total) must fit in ~360px × 640px viewport = ~640px viewport height. Total above-fold height consumed: 34 + 16 + 92 = 142px. Well within fold. Contract is met.

### (c) Wallet Panel States (inline, per-instrument)

The wallet panel is a single component that renders one of 5 states. Each state has distinct affordance. Dimensions: full-width panel within its column, min-height 80px.

| State | Visual treatment |
|-------|-----------------|
| DISCONNECTED | `bg-surface` panel, `text-text-secondary` prompt text, inline RainbowKit ConnectButton (ochre `bg-accent-default text-bg-canvas`, `h-11` min for 44px touch target) |
| CONNECTING | Same `bg-surface` panel, `Loader2` spinner (animated, `text-status-in-progress`), `text-text-secondary` label. No button. |
| CONNECTED_WRONG_CHAIN | `bg-status-parked/10` panel tint, `AlertTriangle` amber icon, amber text label, "Cambiar red" / "Switch network" button (`bg-status-parked text-bg-canvas`) |
| CONNECTED_READY | `bg-status-pass/10` panel tint, `CheckCircle2` green icon, green label, on-chain position data below |
| UNSUPPORTED_CHAIN | `bg-status-fail/10` panel tint, `XCircle` red icon, red label, explanatory copy. No CTA button. |

**Focus management (DEFI-06):** RainbowKit handles focus trap within the modal. On modal close, focus returns to the ConnectButton element. No custom focus trap. Keyboard: Tab moves through interactive elements. Escape closes modal (RainbowKit native). SR announcement: RainbowKit's modal already announces via `aria-modal` + `aria-label`. WalletPanel state changes announce via `aria-live="polite"` on the state container.

## RainbowKit Theme → Token Mapping (DEFI-01 / CONTEXT.md)

RainbowKit v2 `lightTheme()` / `darkTheme()` override via `theme` prop on `RainbowKitProvider`.

```typescript
// Exact token mapping — executor implements this in app/(defi)/providers.tsx
import { lightTheme, darkTheme } from '@rainbow-me/rainbowkit'

const rbkLightTheme = lightTheme({
  accentColor: 'oklch(0.6 0.08 70)',           // --accent-default
  accentColorForeground: 'oklch(0.97 0.005 80)', // --bg-canvas
  borderRadius: 'medium',                        // 0.5rem matches --radius
  fontStack: 'system',                           // overridden below
  overlayBlur: 'none',
})

// Override CSS variables in RainbowKit modal container:
// --rk-fonts-body → var(--font-plex-sans), system-ui, sans-serif
// --rk-colors-modalBackground → var(--bg-elevated)
// --rk-colors-modalBorder → var(--border-default)
// --rk-colors-modalText → var(--text-primary)
// --rk-colors-modalTextSecondary → var(--text-secondary)
// --rk-colors-actionButtonBorder → var(--accent-default)
// --rk-colors-actionButtonBorderMobile → var(--accent-default)
// --rk-colors-connectButtonBackground → var(--accent-default)
// --rk-colors-connectButtonText → var(--bg-canvas)
```

**ConnectButton label override:** Pass `label="Conectar billetera"` / `label="Connect wallet"` via next-intl `t()` string into the `ConnectButton` component. RainbowKit v2 `ConnectButton.Custom` renderer used to inject the locale-appropriate label while preserving RainbowKit's modal machinery.

**Keyboard / a11y (DEFI-06):** RainbowKit v2 uses `@radix-ui/react-dialog` internally — keyboard nav, focus trap within modal, Escape-to-close, `aria-modal`, and `aria-label` are provided natively. No custom implementation needed. The PhaseS wallet panel does NOT add a second focus trap. On modal close, focus returns to the triggering button (RainbowKit native behavior).

**SR state announcements:** `WalletPanel` wraps its state region in `<div aria-live="polite" aria-atomic="true">` so state transitions (DISCONNECTED → CONNECTING → CONNECTED_READY) announce to screen readers without page reload.
