## Accomplishments

- Removed the panel-wide `aria-live="polite" aria-atomic="true"` from `WalletPanel`'s outer div (DEFI-06 BLOCKER 1 — caused full panel re-read on every state transition)

- Created durable `tests/e2e/wallet-connect-modal.spec.ts` covering keyboard operability, focus-return on all close paths (Escape/X/backdrop), connect-success focus to `<output>`, and announcement text changes

## Decisions Made

- Used `<output>` instead of `<p role="status">` — biome's `useSemanticElements` rule enforces the semantic HTML element; `<output>` has implicit `role=status + aria-live=polite`, which is spec-compliant and more appropriate
- Renamed audit route folder from `_a11y-wallet-check` to `a11y-wallet-check` — Next.js App Router treats folders starting with `_` as private/non-routed (this is a known Next.js convention)
- `NEXT_PUBLIC_E2E` is read directly from `process.env` (not added to the validated env schema) — it's a bypass flag, not an application env var; adding it to the schema would require it in `.env.local` for all devs

### Auto-fixed Issues

**1. [Rule 1 - Bug] wallet-read-only.test.tsx: getByText fails with multiple matches**
- **Found during:** Task 1 (running vitest after WalletPanel change)
- **Issue:** `screen.getByText('Solo lectura')` threw "Found multiple elements" because the new `<output>` sr-only node AND the visible pill `<span>` both contain the label text
- **Fix:** Changed to `screen.getAllByText(...)` + filter by `tagName === 'SPAN'` to target the visible pill specifically
- **Files modified:** tests/unit/wallet-read-only.test.tsx
- **Verification:** All 4 tests in the file pass
- **Committed in:** dd8dcff (Task 1 commit)

**2. [Rule 1 - Bug] wallet-states.spec.ts: `[aria-live="polite"]` assertions need update**
- **Found during:** Task 1 (analyzing existing e2e assertions)
- **Issue:** Existing e2e asserted `[aria-live="polite"]` `.toBeVisible()` — but the new `<output>` node is `sr-only` (not visible) and `<output>` doesn't carry `aria-live` as a DOM attribute (it's implicit)
- **Fix:** Changed to `output.sr-only` `.toBeAttached()` + added assertion that `[aria-atomic]` has count 0
- **Files modified:** tests/e2e/wallet-states.spec.ts
- **Committed in:** dd8dcff (Task 1 commit)

**3. [Rule 3 - Blocking] Audit route folder renamed from _a11y-wallet-check to a11y-wallet-check**
- **Found during:** Task 2 (build verification)
- **Issue:** Next.js App Router private folder convention: `_` prefix excludes folders from routing. The `_a11y-wallet-check` folder was built but the route wasn't registered (not in `next build` route output)
- **Fix:** Renamed folder; updated AUDIT_ROUTE constant in e2e spec from `/_a11y-wallet-check` to `/a11y-wallet-check`
- **Files modified:** app/(defi)/a11y-wallet-check/ (folder rename), tests/e2e/wallet-connect-modal.spec.ts
