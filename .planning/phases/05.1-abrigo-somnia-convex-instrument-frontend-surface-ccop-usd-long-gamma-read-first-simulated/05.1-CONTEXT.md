<decisions>
## Implementation Decisions (LOCKED ‚Äî from the reviewed spec)

### Honesty model (the spine ‚Äî non-negotiable)
- Three provenance tiers, each a pill with **color + icon + text** + an `aria-label` carrying the
  full provenance sentence:
  - `fork-fixture` ‚Äî **test-seeded mock pool, not market data**; hand-authored from
    `CcopUsdcPool.t.sol` (rate ‚âà4000, tickSpacing, seeded liquidity, chunk strike/width, fork block).
  - `spec` ‚Äî specified in abrigo-somnia ROADMAP/Phase-8 plans, **not yet on-chain** (streamia,
    commission, data-cost, residual).
  - `schematic` ‚Äî the payoff curve is an **illustrative shape, not a contract-derived function**.
- There is **NO `fork-verified` tier** ‚Äî nothing here is a verified market/on-chain read.
- Page-level **SIMULADO / SIMULATED** badge above the fold (next to `RiskCallout`, at 360px):
  "Base fork ‚Äî mock pool, sin despliegue."
- Unknown values render em-dash (‚Äî), **never 0**. No fabricated numbers anywhere, including JSON-LD.

### Data sourcing
- `lib/apps/abrigo/fixture.ts` (NEW): hand-authored, **dated**, provenance-tagged values.
  On-chain integer quantities (ticks, liquidity, strike) are **bigint-as-string** ‚Äî never JS
  `number` (precision). `number` only for derived display ratios. Each value has a `note` making
  the fork-test-artifact nature explicit (mock token, synthetic seed, +2000-tick offset, width=2
  alignment). No export script from abrigo-somnia (deferred).

### Payoff
- `lib/apps/abrigo/payoff.ts` (EXTEND): add a **schematic** convex generator returning
  `PayoffPoint[]` (positive-gamma shape, net of premium), clearly labeled schematic at the
  component layer. Existing linear-put generator stays for `live` instruments. Faithful
  Panoptic-V2 closed-form is **deferred** to a later module.

### Cash-flow (backend-correct ‚Äî corrected in review)
- `lib/apps/abrigo/cashflow.ts` (NEW): per `08-RESEARCH` + `LongGammaWrapper.claimResidual` NatSpec,
  **surviving collateral is already net of streamia + commission** (pool share-burn nets them).
  Therefore `residual = max(survivingCollateral ‚àí wrapperMeteredDataCost, 0)`. streamia/commission
  shown as **informational reads**, NOT residual subtractions. `wrapperMeteredDataCost` is Phase 9
  (unbuilt) ‚Üí `spec` tier, em-dash. Source: `08-RESEARCH ¬ßresidual` (NOT `MATH.md`, a sketch).

### Registry / type migration
- `lib/apps/abrigo/instruments.ts` (EXTEND): `AbrigoInstrument` becomes a **discriminated union**
  on `kind`:
  - `{ kind: 'live'; id; name; nameEn; chainId; address; deployedAt; strike; slope }` (existing shape)
  - `{ kind: 'simulated'; id; name; nameEn; chainId; fixtureKey }` (no address/strike/slope)
- Remove the uncommitted `fixture-celo-01` temp fixture. Add one `kind:'simulated'` entry:
  `id:'ccop-usd-long-gamma'`, `chainId: 8453` (Base ‚Äî reused only for viem/wagmi compatibility),
  `fixtureKey:'ccop-usd-long-gamma'`.
- **Every** `strike`/`slope`/`address`/`deployedAt` read site must narrow on `kind` first (must
  compile under the `tsc` pre-commit gate): `page.tsx` (`find()`, `currentPrice`, the
  `PayoffDiagramClient` call), `InstrumentParams.tsx`, `InstrumentJsonLd.tsx`. Simulated entries
  render `address`/`deployedAt` as em-dash or fork block ‚Äî never undefined, never fabricated.

### Components
- `PayoffDiagram.tsx` (EXTEND): new props `data: PayoffPoint[]`, `strikeRef?`, `currentPriceRef?`,
  `ariaLabel` (plain serializable values across the RSC‚Üí`PayoffDiagramClient`‚Üídynamic boundary).
  **FIX the 05-04 live-verification BLOCKER**: resolved fixed `h-*` parent (0-height), curve stroke
  meeting **4.5:1** contrast (current 3.44:1 fail), eliminate hydration #418. `type="monotone"` and/
  or dense sampling so the convex shape doesn't read linear. Curve-aware sr-only table. Labeled
  schematic.
- `CashFlowWaterfall.tsx` (NEW): `<dl>` ‚Äî premium ‚Üí informational streamia/commission ("already
  netted") ‚Üí metered data-cost (em-dash, Phase-9) ‚Üí residual; spec-tier pill; no card-in-card;
  correct reading order (WCAG 1.3.2).
- `ProvenanceBadge.tsx` (NEW): SIMULADO badge + reusable per-field pill (fork-fixture/spec/
  schematic); color+icon+text; non-text contrast ‚â•3:1 (1.4.11).
- `SnapshotPoolPanel.tsx` (NEW): renders fork-fixture pool fields (humanRate, tickSpacing,
  seededLiquidity) each with a provenance pill. `PoolStatePanel.tsx` (live `InstrumentState`) is
  left UNTOUCHED for live instruments ‚Äî no wrong-field shoehorning.
- `InstrumentParams.tsx` (EXTEND): variant-aware; Panoptic params with per-row provenance + "fork-
  test parameters" copy; chain row (8453) co-located with the fork-block qualifier, never bare "Base".
- `InstrumentJsonLd.tsx` (EXTEND): variant-aware; simulated branch emits `simulated:true` +
  provenance and **no `strike`/`slope`/`address`**.
- `WalletPanel.tsx` + `lib/wallet/state.ts` (EXTEND): a `readOnly` path with a dedicated read-only
  status (pill + "sin transacci√≥n ‚Äî fork simulado"), **no switch-network CTA**, and it **cannot
  resolve `CONNECTED_READY`**.
- Detail page (EXTEND): branch on `kind==='simulated'` **before** any aggregator call; build pool

  pass `WalletPanel readOnly`.
