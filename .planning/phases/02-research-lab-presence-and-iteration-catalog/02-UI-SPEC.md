## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | shadcn/ui (Feb 2026 refresh) | STACK.md — Tailwind v4 + React 19 compatible |

| Icon library | lucide-react (already used in StatusPill, AppsDropdown) | StatusPill.tsx, AppsDropdown.tsx |
| Body font | `IBM Plex Sans` (Google Fonts, loaded via next/font) | Phase 2 decision — humanist sans, not on impeccable blocklist |
| Mono font | `IBM Plex Mono` (Google Fonts, loaded via next/font) | Phase 2 decision — for replication hashes, β values, p-values |
| Fallback body | `system-ui, -apple-system, "Segoe UI", sans-serif` | Preserves LCP on first-paint before font loads |
| Fallback mono | `ui-monospace, "Courier New", monospace` | Preserves layout stability |

**Font selection rationale:** IBM Plex Sans is the most legible humanist sans not on the impeccable blocklist (Inter, Geist, Mona Sans, Plus Jakarta are blocked). It carries an institutional, research-paper register coherent with the ∂²Π logo. IBM Plex Mono as the companion mono avoids the "dev blog" connotation of JetBrains Mono while sharing the same optical metrics — important when they appear side-by-side in evidence chains.

## Token Migration: Phase 1 → Phase 2

**CRITICAL:** Phase 1 authored `app/globals.css` with hue 165 (teal-green) as the accent placeholder. Phase 2 MUST replace the accent and neutrals with the LOCKED muted ochre palette. The base neutrals must shift from hue 165° to hue 70° to align with the cream-on-ink register.

**Phase 1 tokens to REPLACE (do not keep):**

```css
/* REPLACE: Phase 1 placeholder — teal-green accent, wrong hue for the lab */
--accent-default: oklch(0.55 0.18 165);   /* teal-green placeholder */
--bg-canvas: oklch(0.985 0.005 165);       /* teal-tinted cream — too cool */
--text-primary: oklch(0.18 0.02 165);      /* teal-tinted ink — too cool */
```

**Phase 2 canonical token set (full replacement):**

```css
/* Neutrals — tinted toward hue 70° (warm ochre/parchment register) */
--bg-canvas:   oklch(0.97 0.005 80);   /* cream canvas — matches ∂²Π logo paper */
--bg-surface:  oklch(0.94 0.008 75);   /* card / panel surface */

/* Accent — LOCKED muted ochre */
```

## Spacing Scale

Multiples of 4 only. Aligns with Phase 1 Tailwind v4 utilities (gap-1 = 4px, gap-2 = 8px, etc.).

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | gap-1 / p-1 | Icon gaps, inline badge padding |
| sm | 8px | gap-2 / p-2 | Compact element spacing, pill padding |
| md | 16px | gap-4 / p-4 | Default element spacing, card inner padding |
| lg | 24px | gap-6 / p-6 | Section content padding |
| xl | 32px | gap-8 / p-8 | Card outer margin, form group spacing |
| 2xl | 48px | gap-12 / py-12 | Sub-section vertical rhythm |
| 3xl | 64px | gap-16 / py-16 | Intra-section vertical rhythm |
| 4xl | 80px | py-20 | Hero vertical padding (panoptic reference: 80–120px) |
| 5xl | 120px | py-[120px] | Max section padding on wide viewport |

**Container:** max-width 1200px, `mx-auto px-4 sm:px-6 lg:px-8` — matches panoptic.xyz reference.

**Touch target minimum:** 44px height on all interactive elements on mobile (WCAG 2.5.5). Use `min-h-[44px]` on nav items, iteration card click targets, copy-to-clipboard buttons.

Exceptions:
- TopNav sticky bar: 48px total height (`py-3` + text = 48px) — already authored in Phase 1, do not change.
- StatusPill: 28px height acceptable — read-only, not a tap target.

Font: IBM Plex Sans (body), IBM Plex Mono (code/evidence).

| Role | Tag | Size | Weight | Line Height | Usage |
|------|-----|------|--------|-------------|-------|
| Display | h1 | 36px (text-4xl) | 600 (semibold) | 1.15 | Page-level hero headlines only — `/` and `/apps/abrigo/iterations` |
| Heading 2 | h2 | 28px (text-3xl) | 600 (semibold) | 1.2 | Section titles (e.g., "Iterations", "About the Lab") |

| Label / UI | span, label | 14px (text-sm) | 500 (medium) | 1.4 | Navigation items, pill labels, metadata lines, captions |

**Impeccable rules enforced:**
- No oversized italic-serif hero (blocked by impeccable detector).
- No gradient text on headings (blocked by impeccable detector).
- All headings use solid `text-text-primary`.
- Body text uses `text-text-primary`; secondary prose uses `text-text-secondary`.
- Muted metadata uses `text-text-muted` — never `text-gray-*` on a colored background.

## Color

Base: cream canvas `oklch(0.97 0.005 80)` — ∂²Π logo paper register, locked.
Accent: muted ochre `oklch(0.6 0.08 70)` — locked 2026-05-12, user signoff.

| Role | OKLCH | ~Hex | Usage |
|------|-------|------|-------|
| Dominant (60%) | oklch(0.97 0.005 80) | ~#f7f3ec | Page backgrounds, `--bg-canvas` |
| Secondary (30%) | oklch(0.94 0.008 75) | ~#ede9e0 | Card surfaces, panels, nav bg on scroll |
| Accent (10%) | oklch(0.6 0.08 70) | ~#a87c3a | See "Accent reserved for" list below |
| Destructive | oklch(0.52 0.19 25) | ~#b94040 | No destructive actions in Phase 2; token exists for CI gate |

**Accent reserved for (explicit list — nowhere else):**
1. Primary CTA button fill (`bg-accent-default text-bg-canvas`) — "Browse iterations", "Read detail"
2. External links and inline anchor underlines on hover
3. Focus rings on interactive elements (`ring-accent-default`)
4. Numbered step labels on `/about` methodology section (01 / 02 / 03 / 04 / 05)
5. Replication hash copy-to-clipboard icon on hover/active state
6. Active nav link indicator (bottom-border or text color only — never fill)
7. `<ReplicationHash>` component border on hover

**Accent is NOT used for:**
- Status pills (they use semantic status colors — separate system)
- Section dividers or card borders
- Headings or body text
- Background fills larger than a single interactive element

**Status color matrix (CROSS-09 — color + icon + text, never color-alone):**

| Status | OKLCH (light) | OKLCH (dark) | Icon | Text label (es-CO) | Text label (en) |
|--------|---------------|--------------|------|--------------------|-----------------|
| PASS | oklch(0.38 0.17 145) | oklch(0.62 0.17 145) | CheckCircle2 | Aprobado | Pass |
| FAIL | oklch(0.40 0.19 30) | oklch(0.62 0.19 30) | XCircle | Rechazado | Fail |
| PARKED | oklch(0.42 0.13 60) | oklch(0.65 0.13 60) | PauseCircle | Pausado | Parked |
| IN_PROGRESS | oklch(0.38 0.16 230) | oklch(0.62 0.16 230) | Loader2 | En progreso | In Progress |

**WCAG AA verification (Phase 1 confirmed):** All four status colors at their declared lightness values achieve ≥ 4.5:1 contrast ratio against `--bg-canvas` (~L 0.97) when rendered as text. Pill backgrounds use 10% opacity tint — effective contrast of text against tinted background is maintained. CI enforces via axe-core.

**Epistemic-equality rule (ITER-02):** All four status pills MUST render at identical dimensions. No status gets a larger pill, bolder weight, or different border treatment than any other. This is a hard constraint — fail is as important as pass.

### New — Phase 2 Builds

#### `<IterationCatalogCard>`

Wraps StatusPill. Renders one row in the iteration catalog grid.

**Composition (top-to-bottom, left-to-right):**
```
[StatusPill]  [Iteration title — H3 — text-text-primary]
              [Key metric: β = +0.137 OR "—" if absent — text-sm font-mono text-text-secondary]
              [Analysis date — text-xs text-text-muted]  [Slug — text-xs text-text-muted]
```

**Constraints:**
- Fixed card height: `min-h-[120px]` — ALL statuses render at identical height. No content overflow causes height difference.
- Border: `border border-border-default rounded-lg` — minimal, no colored side-borders (impeccable anti-pattern 4).
- No card-nested-in-card (impeccable anti-pattern 3).
- Hover: `hover:shadow-sm hover:border-accent-default/30` — subtle elevation, 150ms ease-out. No aggressive shadow.
- Active/focus: `focus-visible:ring-2 focus-visible:ring-accent-default`.
- Click target: entire card is a link (`<a>` wraps card content).
- The β metric renders in `font-mono` to visually separate it from prose — but the card itself is not a code block.
- Icon + text + color on StatusPill — never color-alone.

**States:**
- Loading: Skeleton matching card dimensions — `animate-pulse bg-bg-surface rounded-lg min-h-[120px]`.
- Error: Card renders with "Error al cargar / Error loading" in `text-text-muted text-sm` at card height.
- Empty (no iterations): Page-level empty state (see Copywriting), not card-level.

#### `<EvidenceChain>`

Core component for iteration detail pages. Renders the structural evidence block.

**Composition (tabular, not card-in-card):**
```
Evidence chain
──────────────────────────────────────────────
β estimate    │ +0.137
95% CI        │ [0.092, 0.182]
p-value       │ 1.5 × 10⁻⁸
Sample size   │ N = 847
─────────────────────────────────────────────
Replication   │ [sha256: abc123…] [Copy icon]
              │ → make verify (link)
```

**Constraints:**
- Use `<dl>` / `<dt>` / `<dd>` HTML semantics — accessible name–value pairs, not a table.
- All numeric values render in `font-mono text-sm`.
- Replication hash renders in `font-mono text-xs` with truncation at 16 chars + hover tooltip showing full hash.
- `Copy icon` (ClipboardCopy from lucide-react) shows `text-text-muted` default, `text-accent-default` on hover/active.
- Copy success: 1500ms transient tooltip "Copied!" / "¡Copiado!" — CSS `opacity-0` → `opacity-100` → `opacity-0`, 150ms ease-out.
- No color-only state for copy success — icon changes + tooltip text.
- `make verify` link: `text-accent-default underline-offset-2 hover:underline` — opens in new tab.

## Motion Contract

All motion: CSS transitions only. No Framer Motion in Phase 2. No bounce, no elastic, no spring.

| Interaction | Property | Duration | Easing | Trigger |
|-------------|----------|----------|--------|---------|
| Nav hover | `background-color`, `color` | 150ms | ease-out | `:hover` |
| AppsDropdown open/close | `opacity`, `transform: translateY` | 150ms | ease-out | state toggle (already authored) |
| ChevronDown rotate | `transform: rotate` | 150ms | ease-out | AppsDropdown open state (already authored) |
| Card hover | `box-shadow`, `border-color` | 150ms | ease-out | `:hover` on IterationCatalogCard |
| Copy button hover | `color` | 100ms | ease-out | `:hover` |
| Copy success tooltip | `opacity` | 150ms / 1500ms hold / 150ms | ease-out | clipboard write success |
| Language switcher | `background-color` | 100ms | ease-out | `:hover` |

**No transitions on:** page load, iteration card mount, section scroll-into-view, content hydration. First paint must be instant.

**Prefers-reduced-motion:** All transitions must respect `@media (prefers-reduced-motion: reduce)`. Tailwind's `motion-safe:` prefix handles this. Apply to card hover and dropdown transitions.

## Impeccable Anti-Pattern Enforcement Checklist

The following patterns are BLOCKED by `npx impeccable detect` in CI. The executor must not introduce any of these in Phase 2:

| Anti-pattern | Rule | Phase 2 impact |
|-------------|------|---------------|
| Inter / Geist / Mona Sans as typeface | `overused-font` | Use IBM Plex Sans only |
| Purple-to-blue gradient | `ai-color-palette` | Cream + ink + ochre — no gradients |
| Card nested in card | `nested-cards` | ContributorCard is a row, not a card |
| Side-tab accent border (border-left colored) | `accent-border` | No left-colored borders on any card |
| Icon tile stacked above heading | `icon-tile-stacked-above-heading` | Lead with content, inline icons only |
| Bounce/elastic easing | `bounce-or-elastic-easing` | 150ms ease-out only |
| Gray text on colored background | `gray-text-on-colored-background` | Use hue-tinted text tokens only |
| Pure black background | `pure-black-background` | Dark mode uses oklch(0.13 0.015 70) |
| Hero metric layout (three stat trio) | `hero-metric-layout` | IterationCountTile uses divide-y list |
| Gradient text on headings | `gradient-text` | Solid text-text-primary on all headings |
| "Empower your X with Y" copy | copy detector | Author's voice throughout |
| Oversized italic-serif hero | `oversized-italic-serif-hero` | IBM Plex Sans, no italic-serif |
| Eyebrow chips above H1 | `eyebrow-chips` | No eyebrow element before any H1 |
