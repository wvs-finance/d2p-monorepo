# Phase 03.1 Plan 03: Research Reading Page + Component Map + PaperBridge Summary

The paper-grade `/research/[slug]` reading surface, its six-component anatomy, the sole PaperBridge client island, the hermetic arXiv URL deriver, a Mode-B arXiv fixture, and the reading e2e — with the Plan A KaTeX spike retired.

## What shipped

**Reading page (`app/(lab)/research/[slug]/page.tsx`)** — locale-aware hybrid route:
- Renders its **own single `<main>`** (one main landmark/page), accommodating the Wave-2 change where `app/(lab)/layout.tsx` no longer renders `<header>`/`<main>` and the root `TopNav` is the sole site header.
- `await params` (Next 16 Promise params); `generateStaticParams` enumerates distinct slugs (locale resolved from the `NEXT_LOCALE` cookie via `getLocale`, not a URL segment).
- **Mode A** (`readable_on_site:true` + non-empty body) → `<ResearchArticle>` body at the ~64ch measure + `<ArticleTOC>` (sticky lg+, from `entry.toc`) + header (type/track badges, `YYYY-MM-DD · Author` in Plex Mono, abstract) + `<PaperBridge>` + DS2P Labs / ∂²Π footer.
- **Mode B** (`!readable_on_site && (arxiv_id|pdf_url|external_url)`) → header + abstract + `<PaperBridge arxivAbs=…>` ("Read the full paper on arXiv"), **no on-site `<article>` body**.
- **Fallback** (no body AND no bridge) → `notFound()`.
- `generateMetadata` (OG `article`, canonical `/research/{slug}`) + a `<script type="application/ld+json">` **ScholarlyArticle** (html pre-built as a const string before return).

**Components prop injection (confirmed):** the compiled MDX body resolves `h2`/`h3`/`Figure`/`TheoremBlock`/`Sidenote` **only via the `components` PROP** passed to `MDXRenderer` (`getResearchComponents(locale)`). `useMDXComponents` was **NOT** used — it is a no-op under Velite's function-body output. The smoke build confirmed 24 `.katex` nodes and a rendered body on the pair-d Mode-A route.

**The 6 components + PaperBridge:**
- `ResearchArticle` (RSC) — renders the body through `MDXRenderer` at `max-w-[64ch]`.
- `ArticleTOC` (RSC) — list built from `entry.toc` (the `s.toc()` source); sticky lg+, inline narrow, **never a disclosure/`<details>`**.
- `Sidenote` (RSC) — presentation wrapper for the canonical gfm `[^n]` footnotes (no parallel directive); CSS-floated styling lives in the page's route-scoped `<style>` so the no-JS baseline still renders footnotes.
- `TheoremBlock` (RSC) — consumes the `kind`/`label` hProperties from the existing Plan A `:::theorem` directive; renders a bold ochre text label + full ochre hairline border.
- `Figure` (RSC) — semantic `<figure>`/`<figcaption>` with locale-aware label and a `fullwidth` bleed opt-out.
- `mdx-components.tsx` — single module exporting `getResearchComponents(locale)`; all server components; the PaperBridge client island is intentionally **not** imported here.
- `PaperBridge` (`'use client'`) — the **sole** client island under `components/research/`; BibTeX `<pre>` + copy button (`navigator.clipboard.writeText`, transient tooltip, swallowed errors — ReplicationHash pattern) + arXiv/PDF/DOI external links (`target=_blank rel=noopener noreferrer`).

**arXiv deriver (`lib/research/arxiv.ts`)** — `deriveArxivUrls(id)` → `{ abs, pdf }` via string concat only, **no build-time fetch** (hermetic builds, spec §4).

**Mode-B fixture (`cfmm-microstructure-fixture.{es,en}.mdx`)** — `readable_on_site:false`, `arxiv_id: "2401.12345"`, `doi`, inline `$…$` math in both abstracts, and a static `bibtex` block (es-CO authored first). This un-empties the `cfmm-microstructure` track for the index.

**Spike retirement** — deleted `app/(lab)/research/spike/page.tsx`, `content/research/spike-katex.{es,en}.mdx`, and `tests/e2e/research-math-spike.spec.ts`. The migrated pair-d Mode-A body now exercises the KaTeX render path on the real reading route; the new reading e2e carries the `.katex` / no-`.katex-error` / `KaTeX_Main` / `\tag (1)` assertions. The index page's temporary `spike-katex` skip guard was removed.

## TheoremBlock vs impeccable — reconciliation (flagged)

The plan specced TheoremBlock as an "ochre **left-rule** + text label", but impeccable@2.1.8 flags a one-sided `border-left: Npx solid <color>` as the side-tab AI-tell (it caught the spike's `3px solid` ochre rule in Plan A).

**Resolution:** TheoremBlock uses a **full 4-side ochre hairline border** (`border border-accent-default/40`) — a uniform border on all sides is **not** a side-tab — plus a **bold ochre text label** (`text-accent-default`, "Teorema"/"Theorem"/"Definición"/etc.). This:
- satisfies the plan's `grep border-accent-default` acceptance criterion,
- keeps CROSS-09 (callout encodes **color + a text label**, never color alone), and
- passes `npx impeccable@2.1.8 detect app/` → **exit 0** (verified). The one-sided spike border is gone with the deleted spike route.

The unit test `theorem-block.test.tsx` asserts the **label text** is present (not merely a color class).
