### Root cause of the console errors (filed for Plan B/C)
`app/(lab)/research/layout.tsx` hand-rolls font references at a path that is never served:
- L20–33: two `<link rel="preload" as="font" href="/static/katex/...woff2">` — **preload 404**
- L35–49: inline `@font-face { ... src: url('/static/katex/...woff2') }` overrides intended to
  add `font-display: swap` — **src 404, so the swap override is dead**

Net effect:
1. **2 console errors / 4 wasted round-trips** per render (preload + @font-face, ×2 faces).
2. **The `font-display: swap` LCP optimization the layout comment claims does not apply** — the
   overriding `@font-face` can't load, so glyphs fall back to `katex.min.css`'s `@font-face`
   (no `swap`). The stated 3G/FOUC mitigation is fantasy until the URLs resolve.
3. **The `<link rel="preload">` is doubly wrong**: even with a correct path it would preload a
   URL the page never requests (the page uses the `/_next/static/media/<hash>` URLs), triggering
   a "preloaded but not used" warning.

Fix options: (a) copy KaTeX woff2 faces into `public/static/katex/` so the paths resolve, OR
(b) delete the preload + `@font-face` block entirely and rely on `katex.min.css`'s bundled
fonts (already 200), optionally re-adding `font-display: swap` via a rule that targets the
real Turbopack-hashed faces. Option (b) is simplest and removes all 4 errors.
