# Phase 03.1 Plan 01: Math Pipeline + Render-Path Spike Summary

**One-liner:** Build-time KaTeX via `s.mdx({gfm:false})` + `MDXRenderer` components-prop injection proven under dev-Turbopack AND prod-build e2e, with locale-split glob, theorem-directive plugin, and `next-mdx-remote/rsc` fallback exercised.

---

## What Was Built

This plan executed the GATING math-pipeline spike for the research reading surface. Before any downstream `/research` work could begin, this plan proved:

1. **Build-time KaTeX render path works** — `velite.config.ts` compiles MDX bodies with KaTeX at Velite build time (prebuild/beforeCompile, a Node process — Turbopack-safe by construction). KaTeX HTML is embedded in the compiled function-body string, so zero math JS ships to the browser.

2. **Custom-component injection path works** — `MDXRenderer` was resurrected from git `5056bb7^` and extended with an optional `components` prop. Velite@0.3.1 compiles with `outputFormat:'function-body'` and no `providerImportSource`; custom components resolve ONLY via the `components` prop. A no-prop renderer throws on any body containing custom components.

3. **Locale-split glob settled** — Pattern changed to `research/*.{es,en}.mdx`; `locale` derived from `s.path()` suffix (`.es` → `'es'`, else `'en'`). Both `spike-katex.es.mdx` and `spike-katex.en.mdx` emit correct locale values.

4. **Velite auto-GFM suppressed** — `gfm: false` suppresses Velite's auto-prepend of `remarkGfm` (verified against `velite/dist/index.js` L4982/L4989). Spec §0 remark order `[remarkMath, remarkGfm, remarkDirective, remarkTheoremDirective]` honored exactly.

5. **Theorem-directive plugin** — `remarkTheoremDirective` maps `:::theorem/definition/lemma/proof` → `TheoremBlock` hName so Plan C's component map can render them.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] remark-math single-line `$$...\tag{1}$$` parsed as inline math**
- **Found during:** Task 3 (e2e test `BURN-CLASS GUARD: zero .katex-error` FAILED)
- **Issue:** Single-line `$$\Pi(\sigma_T) \approx \hat{K} \cdot \sigma_T \tag{1}$$` was classified as inline math by remark-math (ambiguous delimiter on same line), causing KaTeX to emit `.katex-error` with "\\tag works only in display equations"
- **Fix:** Changed to multi-line display form (`$$\n...\n$$`) in both `spike-katex.es.mdx` and `spike-katex.en.mdx`. Authoring constraint documented in `docs/03.1-RENDER-PATH.md` for Plans B and C authors.
- **Files modified:** `content/research/spike-katex.es.mdx`, `content/research/spike-katex.en.mdx`
- **Commit:** `fb167c2`

**2. [Rule 1 - Bug] `exactOptionalPropertyTypes` incompatibility in MDXRenderer**
- **Found during:** Task 2 (tsc --noEmit failure)
- **Issue:** `return <C components={components} />` passes `undefined` to `components?:` when `components` is absent; `exactOptionalPropertyTypes:true` forbids this
- **Fix:** Conditional render: `components != null ? <C components={components} /> : <C />`
- **Files modified:** `components/MDXRenderer.tsx`
- **Commit:** `9d07d0b`

## Key Findings Documented in docs/03.1-RENDER-PATH.md

1. **Unified@11 deduplication** — One single `unified@11.0.5` in the entire dep tree (confirmed via `pnpm why unified`). No pnpm overrides needed.

2. **Locale-derivation mechanism** — `s.path()` strips only the last extension. `'research/spike-katex.es.mdx'` → `'research/spike-katex.es'`. Transform uses `.endsWith('.es')`.

3. **Components-prop injection finding** — Velite@0.3.1: `outputFormat:'function-body'` + no `providerImportSource`. Custom components resolve ONLY via the `components` prop. This is Plan C's critical integration point.

4. **rehype-pretty-code dropped from v1** — No fenced code in spike fixtures; Shiki LCP weight deferred to v2.

5. **Multi-line `$$` authoring constraint** — Single-line `$$...\tag{1}$$` ambiguous → use multi-line form for all display equations with `\tag`.

6. **Dev-Turbopack verdict: PASS** — 1 katex-display, 0 katex-error, 1 theorem-block via curl.

7. **next-mdx-remote/rsc fallback: PASS** — `compileMDX` compiles spike fixture to React element. Viable emergency option; not used in v1.

8. **Production-build e2e verdict: ALL 5 PASS** — route 200, .katex present, zero .katex-error, (1) tag text, TheoremBlock visible.
