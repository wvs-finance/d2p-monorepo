## Implementation Decisions (locked — from the verified spec; the spec is authoritative)

### Decomposition — 3 sequenced plans (spec §2)
- **Plan A — Math pipeline + render-path GATING spike.** Install `katex remark-math rehype-katex` (pinned, version-checked vs Velite's bundled mdx/unified majors); add a compiled body to the `research` collection; wire build-time KaTeX; resurrect the RSC `MDXRenderer` (git `5056bb7^:components/MDXRenderer.tsx`, the `new Function(code)(runtime).default` pattern); render ONE display equation and verify under BOTH `next dev --turbopack` AND `next build && next start`, live-verified by Evidence Collector (a `.katex` node present AND NOT `.katex-error`). Also settle the locale-split mechanism. **Nothing downstream builds until Plan A returns ✓.** Documented fallback (also exercised in the spike): `next-mdx-remote/rsc` + `s.raw()`.
- **Plan B — Index + content migration + index i18n.** Server `<Link href="?track=">` segmented filter (NOT nuqs — see FOUND-11 below); `searchParams`-driven server filter; migrate the 3 existing research MDX (split bilingual `## Español`/`## English` bodies into `{slug}.es.mdx`/`{slug}.en.mdx`; add required `track`; close the dead-card by setting the y3 write-up `readable_on_site: true`) — schema change + migration ATOMIC; index-chrome i18n; honest empty-per-track state.
- **Plan C — Reading page + paper-bridge + reading i18n + gates.** `/research/[slug]` reading page (locale-aware, slug×locale `generateStaticParams`); ~64ch measure; TOC from H2/H3 (sticky rail lg+, inline on narrow, never `<details>`); Tufte sidenotes→footnotes (CSS-only); theorem/definition/lemma callouts (ochre left-rule + label); numbered figures; `PaperBridge` (arXiv+PDF+static BibTeX copy — sole client island); reading-page i18n superset (es-CO-first, `docs/copy-review.md` sign-off); manual MathML screen-reader entry recorded in `docs/a11y-audit.md`; Lighthouse LCP gate; Evidence Collector.

### Schema additions (spec §4)
Add to `researchSchema` + the `research` collection in `velite.config.ts`: `track: s.enum(['cfmm-microstructure','abrigo-hedge-design','notes'])` (REQUIRED — adding it breaks `velite build` until the 3 files are migrated → atomic with migration), `readable_on_site: s.boolean().default(false)`, `abstract_es?`/`abstract_en?`, `arxiv_id?` (regex `/^\d{4}\.\d{4,5}(v\d+)?$/`, abs/PDF URLs derived), `pdf_url?`/`doi?`/`bibtex?` (all strings — never `s.coerce.date()`; no new shim Date logic). Locale derived from `*.es.mdx`/`*.en.mdx` filename; body compiled via `s.mdx()`.

### Adopted build-time tooling (spec §0 — research-workflow-verified, MIT, zero client JS)
Wire into Velite `s.mdx()` plugin arrays — remark order: `remark-math → remark-gfm → remark-directive`; rehype order: `rehype-slug → rehype-autolink-headings → rehype-katex → rehype-pretty-code`. `remark-gfm` gives footnotes for free; `rehype-slug`+`rehype-autolink-headings` give TOC anchors; `remark-directive` parses theorem blocks (handler hand-authored); `rehype-pretty-code` only if a piece carries code. `fonttools`/`pyftsubset` build-CLI subsets IBM Plex + KaTeX WOFF2 for the LCP budget. KaTeX opts: `output:'htmlAndMathml'`, `throwOnError:false`.

### Citations DEFERRED to v2
v1 paper-bridge = arXiv link + PDF link + static author-pasted BibTeX (copy button) + DOI link. NO `rehype-citation`, NO `@citation-js` (removes the citeproc/SWC prod-build 500 risk). v2 path documented: `rehype-citation` behind a prod-build gate, fed by `zotero-mcp` (GitHub-verified MIT+Docker).

### Equation cross-refs DEFERRED to v2
KaTeX `\tag{}` emits no linkable id; `\label/\ref` unsupported. v1 renders tags as plain text `(n)` flush-right in ochre. v2 option: a rehype `id="eq-N"` plugin + manual `[Eq. (3)](#eq-3)` links.

### Anti-fishing bright line (CROSS-09 + LAB-05)
Prose mention of a closed result (β, p, CI in running text in a finished memo) = ALLOWED. Reconstructing the per-iteration *exercise UI* (interactive forest plots, gate toggles, sensitivity tables as first-class components) = FORBIDDEN. The FAIL memo (`fx-vol-cpi-closed-fail`) reading page renders at IDENTICAL visual weight to a PASS (no muting, no collapse).

- Shiki theme details (only if code appears).
