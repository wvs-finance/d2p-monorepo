## 1. Why re-scope (the empirical chain, condensed)
Validator settlement never occurs on-chain (`SubcommitteePaid`=0 chain-wide); realized caller cost recovers via trace but largely **recycles within the committee**; the paying population is **~76% committee/operator-funded** and **~99% of real SOMI is one infrastructure bot** (`0xe53ff3d7`, agents = 3% of its activity). **→ Somnia mainnet has no organic agent-payment demand**, so the cost function cannot be estimated organically from Somnia. **Decision:** estimate a canonical agentic cost function from mature donor ecosystems and **transfer** it onto Somnia's class structure (Somnia supplies the settlement mechanics + the posted-price anchor; donors supply the demand/cost structure Somnia lacks).

## 2. What the milestones become — and the epistemic status (B1, both reviewers)
- **M1:** build donor panels and **impute** the canonical cost function onto Somnia's classes. **The deliverable is a forward/structural PRIOR, not a measurement — it is not falsifiable on Somnia until organic demand appears.** Milestone success is defined **solely as donor-held-out calibration** (never "Somnia fit").
- **M2:** demand-process estimation runs on **donor-organic demand** (RENDER organic flow + LLM-API usage), NOT Somnia's synthetic arrivals. **Donor request-counts (ACP 2.97M, OLAS 8.44M) are incentive-elastic priced rows and MUST NOT be used as a demand-quantity series** (handoff guard, m4).
- **M3:** composite hedge on the imputed cost × the SOMI/USD FX leg (FX tail is load-bearing — §5/§6).

## 3. The canonical cost function (estimand) — corrected specification
**Part 1 — organic population-definition filter (COVARIATE-ONLY; fixes the Heckman/outcome-truncation defect, MQA-B1):**
`Pr(organic | class, HHI_agent, sybil_label) = Λ(δ0 + δ1·class + δ2·HHI_agent + δ3·sybil_label)`.
Selection is on **covariates and an EXTERNAL, cost-independent sybil/wash label only** — **`log(amount)` is removed** (selecting on a deterministic function of the outcome truncates the cost distribution). **Identifying premise (load-bearing, stated explicitly):** covariate-only selection is valid ONLY under conditional ignorability `C_call ⊥ organic | class, HHI_agent, sybil_label`. This premise **fails for any donor whose `sybil_label` is itself dust-threshold-derived** — ACP's organic carve-out IS the sub-$0.10 cutoff; OLAS has no external label — there the label re-imports the amount-truncation through the back door, and `HHI_agent` correlates with the low-amount infra-bot mass (a collider risk). Therefore: **(a)** covariate-only selection is permitted ONLY for donors with a genuinely external, cost-independent label (x402 via Artemis real-vs-gamed, Dune 6240463/6060125); **(b)** for threshold-derived-label donors (ACP, OLAS) the **two-part/Tobit censored likelihood is MANDATORY, not optional** — estimate the latent below-threshold mass + report the inverse-Mills selection-correction term, never delete survivors; **(c)** a **pre-registered placebo check** re-estimates the `(p_in,p_out)` slope on the organic subsample vs. the full censored-likelihood fit — a material gap is evidence the back door is open and demotes the donor to cross-check.

**Part 2 — realized cost (conditional on organic):**
`C_call = [ p0(class) + p_in(class)·n_in + p_out(class)·n_out ] · ω(ρ_load) + e + γ_gas·gas·gas_price`, then `C_SOMI = C_call · FX`, with `log(R=n_in+n_out)` a class-conditional heavy-tailed mixture and `ω = 1 + κ·(B − B0)_+` (Little's Law).

**Committee aggregation — cost-multiplier separated from reliability (MQA-M4):**
- **cost** uses a per-donor `redundancy_factor` φ: Somnia φ=`subSize`=3 (redundant attestation: 3 members re-execute the SAME call → 3× per-member cost); RENDER φ=1 (partitioned work, after summing shards); ACP/x402 φ=1 (single evaluator). Pooled per-member cost is normalized to a **single logical call** (÷φ) before transfer.
- **reliability** `R_pass = 1 − (1−ρ)^subSize` is a DISTINCT non-cost quantity (used only for the M2 productivity overlay), never as a cost divisor.

**Wave-C3 gate (mechanically enforced, not author discipline):** every parameter reports `1 − Var_post/Var_prior` against a **pre-registered numeric floor τ** (Wave B1); contraction is certified by **simulation-based calibration (SBC, rank-uniformity)** + a **prior-sensitivity sweep run on the data-identified params too** (pooling / prior-misspecification can fake contraction); any parameter below τ or failing SBC is **mechanically relabelled an assumption** with no credible interval. **Slope-orthogonality gate (M2′):** `(p_in,p_out)` are re-fit with all on-chain rows dropped; if the posterior-mean shift or credible-interval-overlap exceeds a pre-registered tolerance, the slope is declared on-chain-contaminated and on-chain donors are hard-restricted to a **structural zero loading on the slope** (not a hoped-for "unchanged").

## 5. Methodology — hierarchical/partial-pooling transfer (corrected)
1. **Slope off-chain** — fit `E[C_call|class]` + variance from LLM price matrix × HAL distributions (the ONLY identification of the slope).
2. **On-chain calibration** — class fixed-effects + escrow/rebate utilization on ACP/OLAS **post-covariate-censor**; x402 cross-check; on-chain donors structurally restricted to `p0`/escrow (no slope leakage).
3. **Normalization** — `× FX` (as a distribution, §6-D3); resource→single-logical-call via φ then ×`subSize` for Somnia; **`s` fixed by {0.03,0.07,0.10}** — only the class ordering is reported as a (weak) test, not as calibration.
4. **Partial-pooling fit (PyMC)** — donors as groups; Somnia a zero-row held-out group with clamped `p0`; data-identified params pooled, prior-only params flagged per §4; RENDER enters as an **aggregate W-elasticity prior only** (it has NO row-level on-chain panel). **Required sensitivity:** report the imputed slope **with/without the RENDER prior** and **with/without farmed-donor down-weighting**, plus a **leave-one-donor-out (LODO) ablation** with a **pre-registered max-swing threshold** (Wave B1); if the imputed-slope shift across donor-drops exceeds it — or if Somnia sits outside the donor convex hull on the diagnostic — the output is labelled a **scenario, not an estimate**.
5. **Somnia-native terms** — congestion `ω`, burn, rebate stay native + UNVERIFIED-pending-bytecode; reported as assumptions.

## 7. Validation & anti-fishing discipline (operationalized)
- Pre-registration is a **frozen hash-logged artifact** (Wave B1) with committed numbers, written before any estimation touches the data; it is itself subject to this gate.
- **External sybil label** (not on-chain concentration alone) is the ground-truth for organic-ness, with a per-donor confusion matrix; unlabelable donors are cross-check only.
- **Identification audit** (Wave C3) with posterior-contraction per parameter is the primary anti-laundering artifact.
- Held-out-donor calibration is **necessary but explicitly NOT sufficient** for the Somnia extrapolation (Somnia is outside the donor convex hull, in a counterfactual resource-priced regime); the LODO ablation + the "imputation/scenario" labelling carry this.
- The two-reviewer gate runs on this plan and on the estimation spec before any fit is load-bearing.
