## Canonical cost function

TWO-PART CENSORED COST MODEL with multiplicative congestion and separable additive gas, in USD then FX-normalized to SOMI.

PART 1 (organic selection / dust censor):  Pr(organic_job | class, agent_concentration, amount) = Λ(δ0 + δ1·class + δ2·log(amount) + δ3·HHI_agent),  used to mask sub-$0.10 dust and down-weight self-dealing infra-bots before any cost estimation (Λ = logistic).

PART 2 (realized per-call cost, conditional on organic):
  C_call(class, R, ρ_load, g, FX)  =  [ p0(class)  +  ( p_in(class)·n_in  +  p_out(class)·n_out ) ]  ·  ω(ρ_load)  +  e  +  γ_gas·gas_units·gas_price(ρ_load)
all divided/multiplied by FX = SOMI/USD to land in SOMI.

Where the consumption vector is itself random (this is the value-add no on-chain donor supplies):
  log(R) = log(n_in + n_out)  ~  Mixture[ class-conditional Normal(μ(class), σ(class)) ],  σ(class) large (≈30x P99/P50 spread, input-dominated n_in >> n_out).

Congestion multiplier (Little's-Law load-dependent term):  ω(ρ_load) = 1 + κ·(B − B0)_+,  B = arrival_rate · latency.

Per-TASK cost = Σ over trajectory calls of C_call (agents emit many calls/task); productivity overlay Cost-of-Pass v = C_task / R_pass(class, model) prices the subSize replication via R_pass = 1 − (1 − rho)^subSize with rho = (1 − e^{−α·X})(1 − e^{−β·L}).

Resource-priced LIMIT (the regime Somnia runners migrate TO): replace the flat per-class constant with C^c = c_comp·(X + L), c_comp = c_e·(2·params + 2·n_layer·n_ctx·n_attn) (Kaplan FLOPs/token), X = reasoning tokens, L = output tokens.

Somnia's CURRENT stop-gap is the DEGENERATE special case: p_in=p_out=0, p0(class) ∈ {0.03, 0.07, 0.10} SOMI, ω≡1 except via the validator-vote adjuster, R collapsed to subSize.

### Parameters

- p0(class): per-class fixed/posted component — Somnia's stop-gap {json-fetch 0.03, llm-inference 0.07, llm-parse-website 0.10} SOMI; the class fixed-effect anchor
- p_in(class), p_out(class): per-input / per-output-token prices, USD/MTok, provider-posted and dated (output/input ratio ∈ [3,5]); identified from donor layer (A)
- n_in, n_out (≡ resource_consumption R): realized prompt/generation token counts per call, distributions per class from HAL/SWE-bench layer (B); the regressor NO on-chain donor carries
- μ(class), σ(class): class-conditional log-consumption mean and (heavy-tailed, ≈30x) dispersion of R
- ω(ρ_load) congestion multiplier; κ = congestion elasticity, B0 = load threshold, B = arrival_rate·latency (Little's Law) — maps to Somnia's validator-vote 95ms/10-block base-fee adjuster
- FX = SOMI/USD cross-rate ρ (network_price); off-chain CoinGecko/Messari, no native Somnia oracle
- γ_gas, gas_units, gas_price: separable additive settlement gas — Somnia floor 6.16e-10..6.16e-9 USD/gas, congestion-adjusted
- subSize (=3) replication factor; α, β = per-agent reliability-curve params in rho=(1−e^{−αX})(1−e^{−βL}); R_pass=1−(1−rho)^subSize
- perAgentBudget = cap; perMember = median(reported executionCost); rebate = p0(class) − realized C_call (the over-collateralization hedge against σ(class))
- δ0..δ3 (selection), HHI_agent (concentration / self-dealing weight), e (external tool charges), c_comp/c_e/params/n_layer/n_ctx/n_attn (resource-priced-limit architecture terms)

## Transfer mechanism

HIERARCHICAL / STACKED-DONOR TRANSFER with dimensional normalization, in five steps.

STEP 1 — Identify the resource SLOPE off-chain (only place it is identified). From donor layer (A) read the posted (p_in, p_out) price matrix across 300+ models; from layer (B)=HAL fit the class-conditional consumption distributions (μ(class), σ(class)) for (n_in, n_out) on SWE-bench/GAIA trajectories. This yields E[C_call | class] = p0 + p_in·E[n_in|class] + p_out·E[n_out|class] AND its variance — the price_per_unit(class)·R slope that is structurally absent from every on-chain donor.

STEP 2 — Calibrate the on-chain POSTED-PRICE + escrow + rebate behavior at scale on the secondary donor. On Virtuals ACP (Dune query 7434268, ~3M jobs) and OLAS legacy (8.44M requests), after applying the Part-1 selection model (censor sub-$0.10 dust, down-weight 200K-475K-job infra agents / staking Safes), estimate the realized-price class fixed effects, the escrow-deposit-vs-realized-cost ratio, and the v2 deliveryRate/maxDeliveryRate utilization distribution — which IS Somnia's perAgentBudget-cap / median(executionCost) / rebate object. Cross-check against the x402 on-chain twin (layer C, Dune query 7623533) filtered to the ~17-35 real recipients, which is the same per-call escrow-step-price form as Somnia and the named K_D anchor.

STEP 3 — Dimensional normalization to Somnia units. (a) Currency: multiply every USD cost by FX=SOMI/USD (off-chain CoinGecko/Messari; carry FX variance forward — it does NOT transfer away). (b) Resource→subSize: Somnia today meters subSize=3 not tokens, so map the donor's per-call E[C_call|class] onto a SINGLE committee-member executionCost, then scale by subSize; the donor's σ(class) calibrates why minPerAgentDeposit·subSize over-collateralizes. (c) Anchor the class ladder: pin the three class fixed effects so the relative spacing matches Somnia's {0.03, 0.07, 0.10} (ratios 1 : 2.33 : 3.33) — i.e. fit a single global scale s and per-class multipliers m(class) such that s·m(json-fetch)≈0.03, s·m(llm-inference)≈0.07, s·m(llm-parse-website)≈0.10 SOMI, with the donor's E[C_call|class] ordering (token-light < token-heavy < long-context-parse) supplying m(class) and SOMNIA_DRAFT's $0.01-class anchors (The Graph query, x402 scrape) cross-validating s.

STEP 4 — Shared-structural-parameter / partial-pooling estimation. Treat {α, β, κ, B0, σ(class), m(class)} as GLOBAL parameters pooled across donors via a hierarchical (Bayesian) model: each donor (A/B, ACP, OLAS, x402, RENDER-organic-prior) is a group with its own intercept but shares the slope/dispersion/congestion structure; Somnia is a held-out group whose class fixed effects are clamped to {0.03,0.07,0.10} while the pooled slope and congestion terms are imputed (transfer learning — Somnia has no organic rows of its own to fit). RENDER supplies the ORGANIC-demand prior on the W-elasticity so the imputed slope is not contaminated by the incentive-farmed donors.

STEP 5 — Somnia-native terms left explicit (NOT transferred). The validator-vote congestion adjuster ω (95ms/10-block base-fee halving/doubling) and the 50/50 burn/validator split + rebate are calibrated from Somnia docs / the HAL Little's-Law g(load) form ONLY structurally (κ,B0 borrowed), and the gross-fee-as-upper-bound and burn rebate r stay Somnia-native and flagged UNVERIFIED pending the implementation-bytecode read.

## Compatibility risks

- SLOPE IDENTIFICATION GAP: every on-chain donor (ACP, OLAS, x402, Somnia) records realized PRICE as a class step-function, NOT consumption-metered cost — so the price_per_unit(class)·R slope is identifiable ONLY from the off-chain (B) HAL token logs. If the HAL 113GB encrypted trace dump cannot be cleanly resolved to per-TASK (vs per-agent-aggregate) cost (HF viewer was erroring; decrypt pipeline required), the central regressor is UNVERIFIED and the slope reverts to extrapolation.
- DONOR-INHERITS-THE-TRAP: x402 (1:1 real:gamed, payers 1033->79), ACP (45% dust, 200K-475K-job infra bots), OLAS (staking-KPI incentive-farmed) and Bittensor (emission-subsidized) all replicate Somnia's synthetic-demand pathology. Without the Part-1 selection/censor model these re-import the exact bias we are escaping; the censor rule itself is UNVERIFIED against any external sybil audit (inferred from on-chain dust mass + concentration only).
- POSTED-PRICE vs RESOURCE-PRICE REGIME MISMATCH: Somnia's {0.03,0.07,0.10} are fixed stop-gap constants with NO effective-date metadata (docs flagged 'stop-gap', volatile), whereas the donor estimates a metered/behavioral pricing function. The transfer is FORWARD-LOOKING (to Somnia's stated future resource-priced regime), not contemporaneous — the W-elasticity inheritance assumption is an extrapolation, not a fit on Somnia data.
- CONGESTION + BURN/REBATE HAVE NO DATA ANALOGUE: Somnia's validator-vote 95ms/10-block base-fee adjuster and 50/50 burn/validator split + closed-form rebate are unobservable in every donor; only the HAL Little's-Law g(load) supplies a structural template (κ,B0 borrowed, not estimated on Somnia). The exact rebate equation lives in the unverified mainnet impl bytecode (NativeTransferFailed path confirmed, closed form not).
- FX / CROSS-RATE: donors settle in USD/xDAI/USDC (≈ trivially USD) so the unit conversion is degenerate for them, but Somnia needs SOMI/USD with NO native on-chain oracle. FX uncertainty does NOT transfer away — it is imported wholesale and stacks on top of the cost estimate, and the convex-perpetual hedge thesis depends on its stochastics, which no donor informs.
- SUBSIZE SEMANTICS: Somnia subSize=3 is REDUNDANT attestation (3 members re-do the same call), whereas RENDER nodes do PARTITIONED work and ACP/x402 use a SINGLE evaluator. Per-member committee payment maps to donor per-node only after dividing out the redundancy factor; mis-handling double-counts cost 3x.
- DUNE COVERAGE BOUNDARY: the on-chain layers union multiple facilitators/chains (x402x vs PayAI vs CDP; Gnosis vs Base; Solana volume) that live in separate tables; a partial union biases volume and price-tier estimates. RENDER/Akash realized-cost is OFF Dune entirely (native APIs only).

- **cost model:** Per-inference / per-attempt monetary cost is the canonical unit. Realized cost of one agent call/attempt is C = (n_in * p_in + n_out * p_out) + extra_charges, where n_in/n_out are prompt/generation token counts observed in the trace and p_in/p_out are the provider's posted per-token prices for the chosen model (output ~5x input). Per-TASK realized cost sums this over all model calls in the rollout trajectory (agents emit many calls per task). The economic productivity metric layered on top is Cost-of-Pass v(m,p) = C_attempt / R(m,p), the expected $ to obtain one correct solution (1/R expected attempts). Resource-priced variant (the 'runner prices by observed resource consumption' future Somnia points to): compute cost C^c = c_comp * (X + L) with c_comp = c_e * FLOPs-per-token (Kaplan scaling-law form 2*params + 2*n_layer*n_ctx*n_attn), separating internal reasoning tokens X from output tokens L. Charging is per-call, metered ex-post on realized tokens, not a fixed posted price per class.

- **cost-function form:** Three nested canonical forms, increasing structure:
(1) REALIZED / POSTED-PRICE (Cost-of-Pass, directly estimable from HAL traces):
    C_call = p_in * n_in + p_out * n_out + e   (e = external/tool charges)
    C_task = sum over calls k in trajectory of C_call,k
    Productivity: v = C_task / R(task_type, model);  R = empirical pass-rate.
(2) RESOURCE-PRICED / DECOMPOSED (Yang & Zhu 2605.23929 — the 'runner meters resource consumption' regime):
    user cost C^u = c_tok * L ;  compute cost C^c = c_comp * (X + L)
    c_comp = c_e * (2*params + 2*n_layer*n_ctx*n_attn)   [Kaplan FLOPs/token]
    reliability rho(X,L) = (1 - e^{-alpha*X})(1 - e^{-beta*L}); optimal L* via water-filling L*_j = (1/beta_j) log(1 + beta_j/theta), theta = shadow price of token budget.
(3) CONGESTION / QUEUEING-AWARE (Kong et al. 2605.15051):
    per-request service demand D = D_fixed + D_load(B), B = effective batch/load inferred from arrival rate via Little's Law (B = arrival_rate * latency); cost/latency rises with load — the load-dependent term is the congestion multiplier.
Combined donor cost function:
    C(task_type, model) = [ p_in*n_in(task_type) + p_out*n_out(task_type,model) ] * g(load) + e
with g(load) = 1 + kappa*(B - B0)_+ the congestion adjuster. Parameters {p_in,p_out} posted+dated; {n_in,n_out} estimated per task_type from traces; {alpha,beta,c_comp} fit if moving to resource-priced regime; {kappa,B0} fit from load-vs-latency curves.
