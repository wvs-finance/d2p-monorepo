| Role | Token | Value | Usage |

| Dominant (60%) | `--bg-canvas` | cream/parchment | Page background of `/apps/abrigo/cornerstone` |

| Accent (10%) | `--accent-default` `oklch(0.6 0.08 70)` (fills/icons/large) · `--accent-text` `oklch(0.48 0.08 70)` (small text) | muted ochre | RESERVED list below |
| Destructive | `--status-fail` | — | NOT used — there is no destructive action in this phase (see Copywriting) |

**Accent reserved for (10% — explicit, exhaustive):**
1. The primary **Confirm** button fill (the single CTA of the flow).
2. The active/selected preset chip ring (when a chip is chosen / free-text resolves to it).
3. The transcript step-marker dot/number for the **currently streaming** step (reuse Phase-7 stepper's existing active-stage treatment — do NOT introduce a new accent usage).

Accent is NOT used for: provenance pills (neutral by contract), mock sub-label, the Agent-1 trace body, mint card values, or general links/icons.

**Honesty color invariants (binding — spec §0 / §6):**
- `fork-verified` tier renders the NEUTRAL token ONLY (`text-text-muted ring-border-default bg-bg-surface`). It MUST NEVER render `--status-pass` / any green/emerald token. (acceptance grep)
- `testnet-agent` tier is also NEUTRAL; recorded/live sub-state is encoded by ICON swap (Archive=recorded, Radio=live), never by color.
- Status pills (LivenessPill "replaying · mock", provenance pills, mock sub-label) encode **color + icon + text + aria-label**, never color alone (CROSS-09).
- The "mock · no en vivo" sub-label renders as: neutral-token pill + lucide `FlaskConical` icon (LOCKED — not `TriangleAlert`, which wrongly connotes error for a benign mock) + text. Color alone never carries the mock meaning.

## Resolved Layout & Interaction Contract (the four open questions)

### Q1 — Chatbot layout → **SINGLE-COLUMN transcript** (RESOLVED)
- One centered column, max-width **~720px**, on the cream canvas.
- Region order top→bottom: (1) compact intro/title, (2) **transcript** region (grows downward as steps stream), (3) **PromptBox** pinned at the bottom of the column (textarea + preset chips above it).
- Rationale: single column reads unambiguously as a chatbot run; a split prompt/stream pane fragments the "watch it run" narrative the judge is evaluating, and is harder to record cleanly. Each workflow step is a stacked transcript entry (Agent-1 → Agent-2 → mint), 24px gap between entries.
- The Agent-1 entry hosts the server-rendered `DecisionPipelineTrace` (passed as `children`), revealed **all-at-once** within its entry — the per-step streaming is ACROSS entries, never within the trace.

### Q2 — Preset chip set + es-CO-first copy (RESOLVED)
Exactly **two** curated chips (one per real recorded decision) + a free-text textarea that resolves to the nearest preset. Each chip maps to ONE real consensus-verified decision and is labeled, equal visual weight, equal dimensions.

| Chip id | → recordedDecisionId | es-CO copy (primary) | en copy (secondary) | Maps to (real) |
|---------|----------------------|----------------------|---------------------|----------------|
| `infl-surprise-add` | `4083729` | "La inflación en Colombia sorprendió al alza — ¿cómo me cubro?" | "Colombian inflation surprised to the upside — how do I hedge?" | consensus 500 → ADD_LONG_GAMMA / sizeBps 6800 |
| `infl-cooling-reduce` | `4083997` | "La inflación se está moderando — ¿reduzco la cobertura?" | "Inflation is cooling — should I reduce the hedge?" | consensus 900 → REDUCE / sizeBps 568 |

- Caption under the chip row (both locales): es-CO "Ejemplos curados · cada uno reproduce una decisión registrada y verificada por consenso" / en "Curated examples · each replays a recorded, consensus-verified run."
- Free-text caption: es-CO "Tu texto se asocia al ejemplo más cercano; no se infiere en vivo." / en "Your text maps to the nearest example; it is not inferred live." (§0.6 — never claim live inference.)

### Q3 — Confirm-card visual treatment + "awaiting confirm" state (RESOLVED)
- `HedgeDecisionCardV2` reuses `HedgeDecisionCard`'s `DataRow` / badge / `CARD_CLASS` idiom (full `--bg-surface` card, 24px padding, hairline `--border-default`, 8px radius). **No `<details>` anywhere in the card** — all ~10 `HedgeLegParamsView` fields render at full visual weight, full prominence (CROSS-09 / LAB-05). No card-nested-in-card (`impeccable`).
- Header carries the `fork-verified` ProvenancePill + the **"mock · no en vivo" / "mock · not live"** sub-label (neutral pill + icon + text).
- Each mock numeric (strike 4.100, size, vol→width, max-loss=premium, margin) renders with an **adjacent sibling unit/`ilustrativo` label** in the same DataRow (e.g. value `4.100` + sibling `mock · cCOP/USD`). max-loss labeled "= prima (máx. pérdida)"; upside labeled "ilimitado (ilustrativo)". No bare `$` as realized PnL.
- **Awaiting-confirm state:** when RunState reaches `a2_decision`, the card shows a primary **Confirm** button (ochre fill, ≥44px) at the card foot, preceded by a one-line gate caption: es-CO "Acción simulada — no se ejecuta en ninguna cadena." / en "Simulated action — nothing executes on-chain." On entering this state, **focus moves to the Confirm button** (a11y — success criterion 2). The button is the ONLY accent fill in the card.
- After confirm, the card stays rendered (no collapse); the mint entry appends below.

### Q4 — Mock step pacing for a clean demo recording (RESOLVED)
Deterministic timed mock producer (`WorkflowEngine`), tuned for a ~10–12s legible recording. Each step entrance uses `motion`, mounted-gated (no non-idle content at SSR), `prefers-reduced-motion` → instant CSS fallback.

| Step transition | Delay before emit | Entrance anim |
|-----------------|-------------------|---------------|
| `idle` → `a1` (Agent-1 reveal) | 600ms after startRun | 250ms fade+rise (16px) |
| `a1` → `a2_decision` (mock card) | 1800ms (let the judge read the real trace) | 250ms fade+rise |
| `a2_decision` → `minting` | on user Confirm (no auto-delay — user-gated) | 200ms fade |
| `minting` → `done` (mint card) | 1200ms (mock tx "settling") | 250ms fade+rise |

- Transcript live region: container `aria-live="polite"` `aria-atomic="false"`; each new step announced exactly once (success criterion 1). Step markers (reuse Phase-7 stepper) are equal-weight; only the active marker uses accent.
- LivenessPill renders a **"replaying · mock" / "reproduciendo · mock"** state during streaming (neutral pill + icon + text, never green, never "live").
