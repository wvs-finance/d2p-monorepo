  struct Request (15 fields, in SOURCE order):
    0 uint256   id
    1 address   requester
    2 address   callbackAddress
    3 bytes4    callbackSelector
    4 address[] subcommittee
    5 Response[] responses          <-- field index 5

    8 uint256   threshold

    11 ResponseStatus status        (enum → uint8)

    13 uint256  remainingBudget
    14 uint256  perAgentBudget
  struct Response (6 fields, in SOURCE order):
    0 address  validator
    1 bytes    result
    2 ResponseStatus status         (enum → uint8)
    3 uint256  receipt              <-- field index 3; a RECEIPT id, NOT a cost; mis-reading this as the cost is the Pitfall-3 trap
    4 uint256  timestamp
    5 uint256  executionCost        <-- field index 5; self-reported, capped at perAgentBudget

  ResponseStatus enum: Pending(1), Success(2), Failed(3), TimedOut(4).
  RequestFinalized(uint256 requestId, ResponseStatus status) fires for ALL terminal statuses
  (Success / Failed / TimedOut). A Failed or no-response TimedOut finalized request legitimately
  has Σ responses[].executionCost == 0 — so a STRICT aggregate `0 < Σ` bound would FALSE-REJECT a
  legitimate finalized request. The synthetic fixture below is a Success request (Σ > 0) used ONLY
  for the structural mis-slice tripwire; the non-strict real-return bound is a 03-04 concern.

  eth_abi tuple type for Request:
    "(uint256,address,address,bytes4,address[],(address,bytes,uint8,uint256,uint256,uint256)[],uint256,uint256,uint256,uint256,uint256,uint8,uint8,uint256,uint256)"
  The aggregate is: sum(r[5] for r in request_tuple[5])   # Σ responses[].executionCost
