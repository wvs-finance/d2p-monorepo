<objective>
Prove WRAP-03's read-from-contract streamia: the wrapper exposes a `recordStreamia()` view that reads `longPremium` from `getAccumulatedFeesAndPositionsData` (never a per-block constant) and reverts `WrongState` when called off the `Open` state, and a fork test swaps fees through the pool (via the Plan-02 `V4SwapHelper`) and proves the read is faithful via a FOUR-part provable proof — (a) READ-FIDELITY: `recordStreamia()` returns EXACTLY the getter's two slots (`longPremium.rightSlot()`/`leftSlot()`) from the SAME call at the SAME tick (wei-exact, always true — this IS the "READ, never re-derived" proof); (b) STATE-GATE: `recordStreamia()` reverts `WrongState` pre-`Open` and post-`Claimed`; (c) NON-ZERO FLOOR: after swaps `recorded0>0 || recorded1>0`; (d) DIRECTIONAL/MONOTONIC: more swaps => recorded strictly increases. Tree-before-impl.

Purpose: This is the make-or-break P1 proof — streamia READ, never re-derived. The proof is read-fidelity (recorded == the getter's own slots, same call/same tick), NEVER read-vs-formula (RESEARCH Pattern 3). Because the equality alone cannot, by itself, distinguish a wrapper that READS the correct long slot from one that reads `shortPremium` or applies a multiplier, the discrimination is carried by THREE corroborating guards the plan pins explicitly: (1) the test pins the CORRECT-tokenId `longPremium` slot on the RHS (a `shortPremium`-reading or wrong-leg wrapper diverges from this pinned RHS), (2) the non-zero + monotonic checks (a constant/multiplier wrapper fails directional), and (3) the P1 grep-guard (no `SPREAD_MULTIPLIER`/`perBlock` constant in src or test). The previously-planned cross-tick `assertEq(recorded, OptionBurnt.premiaByLeg)` is REMOVED as a gate: `recordStreamia` reads `longPremium` at `currentTick` (PanopticPool.sol L437/L445), but `OptionBurnt.premiaByLeg` is emitted from `_burnOptions`->`_updateSettlementPostBurn` with `COMMIT_LONG_SETTLED=true` => `_getPremia` atTick=0 (L1159-1161), and `getFeeGrowthInside` branches on the passed tick — so `recorded(currentTick) != premiaByLeg[0]` in general (the available-premium cap at L2084-2122/L1186-1235 is applied ONLY in the SHORT branch, so it is NOT the cause; the long leg keeps the raw `_getPremia` in both paths — a wei-exact `assertEq(rec_i, settled_i)` would simply revert). Swaps (not block advance) move `feeGrowthInside` so premium becomes observable (Pitfall 2). This stays within CONTEXT §Discretion B3, which already named the directional fallback — no CONTEXT change.
Output: `streamia.tree` (committed first), `recordStreamia()` view (with the `WrongState` state-gate) in `LongGammaWrapper.sol`, `streamia.t.sol` green on the fork.
